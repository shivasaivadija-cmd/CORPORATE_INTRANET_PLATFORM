================================================================================
TECHNOLOGY STACK
================================================================================

Frontend Technologies:
  - Framework: Next.js 15 (App Router)
  - Language: TypeScript 5.7
  - Styling: Tailwind CSS 3.4
  - UI Components: Radix UI + Custom Components
  - Animations: Framer Motion
  - State Management: Zustand + TanStack Query (React Query)
  - Icons: Lucide React
  - Form Validation: Zod

Backend Technologies:
  - Framework: NestJS 10
  - Language: TypeScript 5.7
  - ORM: Prisma 5
  - Validation: class-validator, class-transformer
  - API Documentation: Swagger/OpenAPI 3.0
  - Authentication: JWT + Refresh Tokens
  - Authorization: Role-Based Access Control (RBAC)

Database & Caching:
  - Primary Database: PostgreSQL 16
  - Cache Layer: Redis 7
  - Job Queue: BullMQ
  - Search: PostgreSQL Full-Text Search
  - ORM: Prisma with migration support

Real-time & AI:
  - WebSocket: Socket.IO
  - AI Provider: Grok AI (xAI)
  - AI Model: grok-2-1212
  - Embeddings: text-embedding-004

Infrastructure:
  - Containerization: Docker + Docker Compose
  - Reverse Proxy: Nginx
  - CI/CD: GitHub Actions
  - Monorepo Tool: Turborepo
  - Cloud Storage: AWS S3 / Cloudflare R2

Mobile:
  - Framework: React Native (Expo)
  - Navigation: Expo Router
  - State Management: Zustand + TanStack Query

PROJECT ARCHITECTURE
================================================================================

Monorepo Structure:
  corporate-intranet-platform/
  ├── apps/
  │   ├── api/                    # NestJS Backend API (Port 4000)
  │   │   ├── prisma/            # Database schema, migrations, seeds
  │   │   └── src/
  │   │       ├── common/        # Shared utilities, guards, decorators
  │   │       ├── config/        # Configuration modules
  │   │       ├── database/      # Prisma service
  │   │       └── modules/       # Feature modules (Auth, Users, Feed, etc.)
  │   ├── web/                    # Next.js Frontend (Port 3000)
  │   │   └── src/
  │   │       ├── app/           # App Router pages
  │   │       ├── components/    # React components
  │   │       ├── hooks/         # Custom React hooks
  │   │       ├── lib/           # Utilities & API client
  │   │       └── store/         # Zustand stores
  │   └── mobile/                 # React Native Mobile App
  │       └── src/
  │           ├── app/           # Expo Router screens
  │           ├── components/    # Mobile components
  │           └── lib/           # Mobile utilities
  ├── packages/
  │   └── types/                  # Shared TypeScript types
  ├── infrastructure/
  │   ├── docker/                 # Docker configurations
  │   └── nginx/                  # Nginx configurations
  ├── docs/                       # Comprehensive documentation
  └── scripts/                    # Build & deployment scripts

System Architecture:
  Client Layer (Web/Mobile) → API Gateway (Nginx) → NestJS Application
  → PostgreSQL Database + Redis Cache + Grok AI API

- Guard-based route protection

DEPLOYMENT CONFIGURATION
================================================================================

Environment Variables:

Backend (apps/api/.env):
  NODE_ENV=production
  PORT=4000
  API_URL=https://api.yourdomain.com
  FRONTEND_URL=https://yourdomain.com
  DATABASE_URL=postgresql://user:pass@host:5432/db
  REDIS_HOST=redis-host
  REDIS_PORT=6379
  REDIS_PASSWORD=redis-secret
  JWT_SECRET=<64-char-secret>
  JWT_EXPIRES_IN=15m
  REFRESH_TOKEN_SECRET=<64-char-secret>
  REFRESH_TOKEN_EXPIRES_IN=7d
  GROK_API_KEY=<your-grok-api-key>
  GROK_MODEL=grok-2-1212
  AWS_REGION=us-east-1
  AWS_ACCESS_KEY_ID=<your-key>
  AWS_SECRET_ACCESS_KEY=<your-secret>
  S3_BUCKET_NAME=intranet-uploads

Frontend (apps/web/.env.local):
  NEXT_PUBLIC_API_URL=https://api.yourdomain.com/api/v1
  NEXT_PUBLIC_WS_URL=https://api.yourdomain.com

Deployment Options:
1. Docker Deployment (Recommended)
   - Use docker-compose.prod.yml
   - Configure environment variables
   - Run database migrations
   - Start all services

2. Vercel (Frontend Only)
   - Deploy Next.js app to Vercel
   - Configure environment variables
   - Connect to external API

3. Railway/Render (Backend)
   - Deploy NestJS API
   - Provision PostgreSQL and Redis
   - Configure environment variables

4. AWS/GCP/Azure (Full Stack)
   - Use container orchestration (ECS, Cloud Run, AKS)
   - Set up managed databases
   - Configure load balancers
   - Implement auto-scaling
